# MVVM
WPF中的MVVM-DEMO

MVVM （Model-View-ViewModel的简写）我的理解就是对UI和代码进行弱绑定。
可以从百度搜到一大堆的解释，

##总结起来，
##MVVM的特点&优点：
##1.UI和业务代码分离,
##  View指的是XAML布局文件,
##  ViewModel相当于原本的xxx.cs文件
##  Binding相当于 xxx.Text = "xxx"
##  Command相当于Click
##  当你修改XAML的时候根本不会影响到ViewModel中的代码和逻辑。
##  当你修改ViewModel.cs的时候也不会因为xxx_Click 而去修改XAML中的控件而心烦
